The problem

A green eval only means something if you defined what red looks like.

Teams ship agents that work in the demo and quietly break in production: the model picks the wrong tool, leaks its prompt to a clever user, confirms an action it should have refused, or loops. Most agent "evals" pass because nobody wrote the test that would have failed. Gauntlet is the part that goes looking.

!

Prompt & secret leaks

Injection, fake system-role escalation, echo tricks, multilingual wrappers — does your agent spill its system prompt or keys?

⤬

Unsafe actions

False-premise refund confirmations, high-blast-radius tool calls, authority-claim social engineering. Did it do the thing it should have refused?

∞

Crashes & runaway output

Oversized payloads, control chars, contradictory constraints, loop bait. Does it 500, hang, or blow past a sane length?

How it works

Four stages. Framework-agnostic.

If your agent speaks HTTP, Gauntlet can test it. Point --target at any endpoint that takes JSON and returns text or JSON.

01

Adversaries

A deterministic library of probes across injection, scope, false premises, exfiltration, malformed input, and loop bait. Reproducible by design.

02

Runner

Fires probes concurrently at your HTTP endpoint. Stdlib-only, no SDK lock-in.

03

Graders

Universal reliability checks plus your canaries — the things your agent must never output or do — ranked CRITICAL to INFO.

04

Report

A readable summary, the worst failures, and a JSON artifact. Nonzero exit gates your CI.

Why Gauntlet

Find the failures. Make sure they stay fixed.

CI

Gate in CI

One command, nonzero exit on anything at or above your threshold. Drop it straight into GitHub Actions.

∆

Regression tracking

The hosted dashboard stores every run and diffs against your last green one — catch the day a fix silently regresses.

⚑

Define your canaries

A small JSON says what your agent must never output and must never comply with. Honest evaluation starts with naming red.

⚙

Any framework

LangChain, custom, hosted — doesn't matter. HTTP in, JSON out. Configurable request/response fields.
